# ADB Reconnaissance — 2026-05-28

Raw findings from live ADB probing of stock Android 8.1.0 on the
Lenovo ThinkSmart View CD-18781Y (serial `HUA0AEFD`).

All commands were run as the unprivileged `shell` user; SELinux enforcing blocks
dmesg and many uevent/modalias reads.

---

## I2C Bus Topology

| Bus   | QUP Controller   | Devices                        |
|-------|-----------------|-------------------------------|
| i2c-1 | `0x78b5000.i2c` | `1-0049` TAS5782M audio amp   |
| i2c-3 | `0x78b7000.i2c` | `3-0014` (no driver), `3-0038` (no driver), `3-0048` Himax touchscreen |

Device tree node names under `/proc/device-tree/soc/i2c@78b7000/`:

- `gt9xx-i2c@14`
- `focaltech@38`
- `himax_ts@48`

Conclusion: addresses `0x14` and `0x38` are touchscreen fallback population options,
not proximity/ALS sensors on this unit.


### Confirmed SSC Config Block Details

From `sensor_def_qcomdev.conf` lines around the VCNL4200 section:

- `VCNL4200` is configured as `SSI SMGR Cfg 1`
- `i2c_bus = 2`
- `i2c_address = 0x51`
- `gpio1 = 43` (ALS/PRX interrupt)
- `sensor_id = 40`
- `data_type1 = 5`, `data_type2 = 6`

Same hardware section also includes:

- `BMA253` accelerometer (`SSI SMGR Cfg 0`)
- `i2c_bus = 2`
- `i2c_address = 0x18`
- `gpio1 = 42`

Runtime naming note:

- `dumpsys sensorservice` reports `BMA255 Accelerometer/Temperature/Double-tap`
- static conf comment labels this block as `BMA253`
- this appears to be a BMA25x family naming mismatch rather than a clear contradiction

SSI GPIO map snippet in this section:

- `I2C SDA_1 = 6`
- `I2C SCL_1 = 7`
- `ALS/PRX INT = 43`

## Sensor Runtime Validation (`dumpsys sensorservice`)

Raw capture saved at:

- `research/sensors/inventories/dumpsys-sensorservice-2026-05-28.txt`

Confirmed runtime sensors on stock Android:

- `BMA255 Accelerometer/Temperature/Double-tap` (BOSCH)
- `VCNL4200 PRX` (Capella)
- `VCNL4200 ALS` (Capella)

Important validation points:

- `Sensor Device: Total 15 h/w sensors, 15 running`
- Proximity and ALS are both explicitly exposed as `VCNL4200` by name
- Recent sensor events show changing VCNL4200 ALS lux values, confirming live data path
- Active clients include Android automatic brightness controller using VCNL4200 ALS

This confirms the stock Android proximity/light path is VCNL4200 over SSC,
not the unbound `i2c-3` touch fallback addresses.

## Camera Runtime Validation (`dumpsys media.camera`)

Raw capture saved at:

- `research/camera/inventories/dumpsys-media-camera-2026-05-28.txt`

Confirmed runtime camera properties on stock Android:

- `Number of camera devices: 1`
- Camera 0 facing: `Front`
- Orientation: `270`
- Flash: `false`
- Focus mode: `fixed`
- Focal length: `4.04`
- Horizontal / vertical FOV: `64.8 / 51.2`
- Active array: `2608 x 1960`
- Pixel array: `2624 x 1976`
- Bayer pattern: `GRBG`

These runtime values align with the front-camera profile from `camera_config.xml`
(S5K4E8/S5KC505A front candidate entries) and with the known front-only exposed camera
behavior in the current firmware.
